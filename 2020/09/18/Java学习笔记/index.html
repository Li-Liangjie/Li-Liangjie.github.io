Java面向对象编程

C语言是面向过程开发的代表，而像C++或者是Java是面向对象编程的语言。

面向过程指的是面对于一个问题的解决方案，更多情况下是不会做出重用的设计思考的，而面向对象的主要设计形式为 模块化设计 ，并且可以进行重用配置。在整个的面向对象的设计里面更多情况下考虑的是标准，在使用的时候根据标准进行拼装，面向对象设计三个主要的特征：

· 封装性 ：内部的操作对外部而言不可见，当内部的操作都不可直接使用的时候才是安全的；

· 继承性 ：在已有结构的基础上继续进行功能的扩充；

· 多态性 ：是在继承性的基础上扩充而来的概念，指的是类型的转换处理。

面向对象程序的开发之中一般三个步骤

· OOA ：面向对象 分析 ；

· OOD ：面向对象 设计 ；

· OOP ：面向对象 编程 。

面向对象核心组成是：类与对象

类与对象简介

类是对某一类事物的共性的抽象概念，而对象描述的是一个具体的产物。如“人类和具体的人”、“汽车设计图纸和一辆真实的汽车”……

类是一个模版，而对象才是类可以使用的实例，先有类再有对象。

在类之中一般都会有两个组成：

·成员组成（Field）：有些时候为了简化称其为 属性 ；

·操作 方法 （Method）：定义对象具有的处理行为；

成员属性封装

在类之中的组成就是属性和方法，一般而言 方法 都是对外服务的，所以 不会进行封装 ，而对于属性由于其需要较高的安全性，所以往往需要对其进行保护，这个时候就需要采用封装性对属性进行保护。

在默认的情况下，对于类中的属性是可以通过其他类利用对象进行调用的，如果想解决这样的问题可以利用private关键字对属性进行封装处理。

如果要想让外部的程序可以访问封装的属性，则在Java开发标准中提供有如下要求：

setter、getter设置或取得属性可以使用setXxx()、getXxx()，以对象属性private String name为例

构造方法

现在的程序在使用类的时候一般都按照了如下的步骤进行：

·声明并实例化对象，这时候实例化对象中的属性并没有的数据存在，都是其对应数据类型的默认值；
·需要通过一系列的setter方法为类中属性设置内容

范例:

在Java中为了考虑到对象的初始化的问题，专门提供有构造方法，即：可以通过构造方法实现实例化对象中的属性初始化处理 。 只有在关键字 new 的时候使用构造方法 ，在Java程序中构造方法的定义要求如下：

·构造方法名称必须与类名称保持一致；
·构造方法不允许设置任何返回值类型，即：没有返回值定义；（但是不能加void，否则会被程序认为是普通方法）
·构造方法是在使用关键字new实例化对象的时候自动调用的。

范例：定义构造方法

当前的对象实例化格式与之前的对象实例化格式的比较：

    ·之前的对象实例化格式：①Person ②A = ③new ④Person()；
    ·现在的对象实例化格式：①Person ②A = ③new ④Person("张三", 11)；
        |-“①Person”：主要是定义对象的所属类型，类型决定了你可以调用的方法；
        |-“②A =”：实例化对象的名称，所有的操作通过对象来进行访问；
        |-“③new”：开辟一块新的堆内存空间；
        |-“④Person("张三", 11)”：调用有参构造；“④Person("张三", 11)”：调用无参构造。

在Java程序里面考虑到程序结构的完整性，所有的类都会提供有构造方法，如果现在类中没有定义任何的构造方法，那么一定会默认提供有一个无参的、什么都不做的构造方法，这个构造方法是在程序编译的时候自动创建的。如果在类中明确定义有一个构造方法的时候，那么这个默认的构造方法将不会被自动创建。

结论：一个类至少存在有一个构造方法，永恒存在。

构造方法也是方法，而 方法可以重载 ，因此构造方法也可以重载。

范例：构造方法重载

在进行多个构造方法定义的时候可以按照参数的个数 升序 或 降序 排列。

构造方法可以进行数据的设置，而对于setter也可以进行数据的设置。构造方法是在对象实例化的时候为属性设置内容的初始化内容，而setter除了拥有设置数据的功能之外还具有修改数据的功能。

This关键字

在程序里面，使用this关键字可以实现以下三类结构的描述：

    ·当前类中的属性：this.属性；
    ·当前类中的方法（普通方法、构造方法）：this（）【构造方法】、this.方法名称【普通方法】；
    ·描述当前对象；

    简单Java类

    在以后进行项目的开发与设计的过程中，简单Java类都将作为一个重要的组成部分存在，在以后的项目中简单Java类无处不在，并且有可能产生一系列的变化。

简单Java类指的是可以表述某一类信息的程序类，例如：描述一个人、描述一个部门、描述一个雇员，并且在这个类之中并没有特别复杂的逻辑操作，只作为一种信息存储的媒介存在。

对于简单Java类而言，其核心的开发结构如下：

  • 类名称一定要有意义，可以明确的描述某一类事物；
  • 类之中的所有属性都必须使用private进行封装，同时封装后的属性必须（ 对简单Java类而言 ）要提供有setter、getter方法；
  • 类之中可以提供有无数多个构造方法，但是必须要保留有无参构造方法；
  • 类之中不允许出现任何的输出语句，所有内容的获取必须反回；
  • 【非必须】可以提供有一个获取对象相信的方法，暂时将此方法名称定义为getInfo
