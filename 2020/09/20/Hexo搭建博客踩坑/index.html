mac OS 10.15操作系统下Hexo搭建博客踩坑

发表于 2020-09-20
本文字数： 4.6k 阅读时长 ≈ 4 分钟

我从2019年的2月开始准备考研。

每年的6、7月份，也就是暑假，被称为考研的”黄金时期”，而我却在别人都在努力学习的时候回家去兼职教书，直接导致了9月开学后我和别人在知识掌握程度上的巨大差距，我很害怕，尤其是我并不擅长的考研数学。

可是9月份当我沉下心去学习的时候，我却发现我在家教书的两个月时间内，逐渐发现了 输出 的重要性。

为什么这么讲？我暑假兼职的是数学老师，很惭愧的说，我自己当年学习高中数学是非常痛苦的，我虽然很用心的去听老师讲、去学习，但仍觉得摸不着头脑。可是当我时隔三年重新回头，作为一个 讲述者 讲授高中数学的时候，我不知不觉的摸清了高中数学的总体框架、做题思路以及技巧，这是我当时作为一个 倾听者 完全没有意识到事情。

因此，我尝试着站在一个讲述者的立场上进行考研数学的学习，虽然最终的成绩并不是很理想，但是我的专业课却由于 输出 的技能而受益匪浅，取得了较好的成绩。

这一周脑子里一直有个想法：建立自己的博客，作为自己 输出 的平台，记录自己学习一门语言、实现一个idea、完成一个项目的过程与心得。

如果我能够将一门语言讲了什么”说的清楚”，那就说明我大概了解了这门语言；

如果我能够将实现一个idea的过程”讲的明白”，那就说明在这个过程中我有过自己独立的思考而非生搬硬套；

如果我能将一个完成一个项目的所有的过程文档”记录下来”，那就说明这个项目我曾经有用心的参与过。

总而言之，我希望能够通过 输出 来提高自己的逻辑思维能力和语言表达能力，毕竟从前面这么废话可以看得出来

——我这两项能力很弱。

那么，就从用Hexo搭建一个自己的个人博客开始吧！

注意：

1、我的博客是在mac OS 10.15操作系统下进行搭建的，因此，可能和Windows操作系统下的搭建步骤有所不同，请注意区分！

2、我的博客的配置以及各种参数的设置，主要是在集成编译软件IDEA（ultimate版）下完成的（当然也可以借助命令行实现相关配置和参数的设置），因此，需要在电脑上预装IDEA（ultimate版）.

Hexo是什么？

Hexo是一个框架，我们可以利用Hexo框架进行我们博客的搭建。

Hexo最初是由Tommy Chen在2012年创建和维护的。自那时以来，它已经帮助成千上万的人建立了他们梦想的网站/博客。

搭建个人博客的两个步骤

大体上讲，搭建自己的博客可以分为两个步骤

1、在本地搭建博客------------------------------>你的博客只能在你的电脑上查看；
2、将博客部署至Github或者Coding分配给你的域名上--->你的博客可以在任何地方通过分配的域名进行访问.

关于一些hexo的常用的命令

hexo clean 这个 不会 删除你的hexo博客， 个人理解 是帮你清除一下”缓存”（表述不一定准确）之类的，重新布置了你的博客之后可以先clean一下，可以解决大部分的 修改博客布置不生效 的问题；

hexo g，是hexo generate的缩写， 个人理解 是相当于让你的博客的布置生效；

hexo s，是hexo server的缩写， 个人理解 是相当让你的博客开启 本地运行 ，你可以通过localhost:4000本地访问你的博客。

hexo d，是hexo deploy的缩写， 个人理解 是相当于把你的博客部署至你的域名上，这样你在本地对博客进行的修改就会在域名下生效，别人就可以通过访问你的域名看到你更改后的博客。

参考： https://zhuanlan.zhihu.com/p/114195340

简单说一下什么叫”部署”， 个人理解 是将你的网站（就是那个blog文件夹）挂载于某个代码管理平台（如github、coding）上，这个代码平台会为这个文件提供一个绑定的域名，当完成后续步骤的配置后，就可以实现： 你在本地对网站做的修改会以文件的形式同步到代码管理平台，而别人可以通过这个绑定的域名访问你放在代码平台上的文件，在浏览器中显示出来就是你的网站。
